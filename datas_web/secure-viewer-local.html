Local Secure Viewer

KoreanTalk 세션 기반 뷰어 검수 런처

이 페이지는 정적 HTML을 직접 열지 않고, same-origin 개발 서버에서 세션 manifest와 PDF 프록시를 응답하는 구조를 전제로 합니다. 운영 Koreantalk에 붙이기 전, 로컬에서 `?session=` 강제 흐름과 UI를 검수할 수 있습니다.

검수 원칙

`?session=` 없이 접근하면 차단 상태가 보여야 하고, PDF는 반드시 `/api/viewer/session/{token}/pdf`를 통해 로드되어야 합니다.

현재 구조

개발 서버는 `dev/secure_viewer_sessions.json`을 읽고, 샘플 세션별 manifest를 same-origin으로 응답합니다.

운영 연결

운영에서는 동일한 URL 구조를 PHP CodeIgniter 컨트롤러와 실제 구매 검증 로직으로 대체하면 됩니다.

실행 명령

저장소 루트에서 아래 둘 중 하나로 실행합니다.

python3 datas_web/secure_viewer_dev_server.py

./datas_web/start_secure_viewer_dev.sh
